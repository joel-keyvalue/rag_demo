# 1. What Is an Embedding?

An **embedding** is a numerical representation of meaning.

Example text:

Fix broken deployment

Embedding representation (simplified):

[0.12, -0.44, 0.81, 0.33, ...]

This list of numbers is called a **vector**.

Each number represents a coordinate in a **high‑dimensional semantic space**.

Key property:

**Similar meaning → vectors close together.**

---

# Where Does the Array Come From?

The array is generated by an **embedding model**.

Example pipeline:

Text  
→ Tokenization  
→ Transformer layers  
→ Projection layer  
→ Embedding vector

Example using Python:

```python
from openai import OpenAI

client = OpenAI()

response = client.embeddings.create(
    model="text-embedding-3-small",
    input="Fix broken deployment"
)

embedding = response.data[0].embedding
print(embedding[:10])
```

The model outputs something like:

```
[0.012, -0.442, 0.812, 0.331, ...]
```

Typical embedding sizes:

| Model | Dimensions |
|------|-------------|
text-embedding-3-small | 1536
text-embedding-3-large | 3072

# 2. Understanding Embedding Length (Vector Magnitude)

Each embedding is an **arrow in high‑dimensional space**.

Two important properties:

1. Direction  
2. Length

Visualization:

Short vector

→

Long vector

────────→

Length is called **vector magnitude**.

Formula:

||v|| = √(v1² + v2² + v3² + ...)

Example:

v = [3,4]

Length = √(3² + 4²) = 5

---

# Why Length Matters

Some similarity metrics consider vector length, others ignore it.

Direction represents **semantic meaning**.

Length represents **strength of the signal**.

But in semantic search we usually care about **direction only**.

---

# 3. Cosine Similarity

Cosine similarity measures the **angle between two vectors**.

It ignores vector length.

Visualization:

Query →

Document →

Small angle → high similarity

Range:

1 → identical meaning  
0 → unrelated  
-1 → opposite

Typical semantic search:

0.8+ strong match  
0.6 moderate match

---

# 4. Dot Product

Dot product measures:

direction similarity × magnitude

Visualization:

Vector A →→→

Vector B →→→

Aligned long vectors → large score.

If vectors are normalized (length = 1):

dot product ≈ cosine similarity.

Many vector databases use dot product because it is **faster to compute**.

Examples:

- FAISS
- Pinecone
- Qdrant
- Milvus

---

# 5. Euclidean Distance

Euclidean distance measures **physical distance between points**.

Visualization:

Point A *
        \
         \ distance
          \
           * Point B

Small distance → similar  
Large distance → different

Less common for text embeddings.

---

# Comparison Summary

| Metric | Measures | Best Use |
|------|------|------|
Cosine similarity | vector direction | semantic search |
Dot product | direction + magnitude | vector DB optimization |
Euclidean distance | physical distance | clustering / ML |

# 7. Where Embeddings Are Stored

Embeddings are stored in **vector databases**.

Examples:

- Qdrant
- Pinecone
- Weaviate
- Milvus

PostgreSQL alternative:

pgvector

Example:

```sql
CREATE TABLE documents (
  id SERIAL,
  content TEXT,
  embedding VECTOR(1536)
);
```

Query example:

```sql
SELECT *
FROM documents
ORDER BY embedding <-> query_vector
LIMIT 5;
```

# Key Takeaway

Think of embeddings as **arrows in space**.

Direction → meaning  
Length → strength

Cosine similarity compares direction,  
Dot product compares direction + magnitude,  
Euclidean distance compares physical distance.
